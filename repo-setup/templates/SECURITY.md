# Security Policy

## Supported Versions

Only the latest release branch receives security updates.

## Reporting a Vulnerability

SonarMD takes security seriously. If you discover a vulnerability, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

### How to report

Email **security@sonarmd.com** with:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### What to expect

- **Acknowledgment** within 2 business days
- **Assessment** within 5 business days
- **Resolution timeline** communicated after assessment
- Credit in the fix (if desired) after the patch is released

### Scope

This policy covers all SonarMD repositories under the `sonarmd` GitHub organization.

### Compliance

SonarMD is a HIPAA-covered entity. All security issues are evaluated for PHI impact and reported per our incident response plan and BAA obligations.
